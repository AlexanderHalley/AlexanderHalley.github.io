Autonomous Lunar Rover

Northwestern Space Technology And Rocketry Society (NUSTARS) | Software Lead

Overview

Leading autonomous navigation development for Northwestern's entry in NASA's Lunabotics competition. The rover must autonomously navigate simulated lunar terrain, excavate regolith, and deposit it in designated zones—all without human intervention and without GPS.

The competition simulates the challenges of lunar construction: fine abrasive dust, communication delays, and the need for full autonomy. Our approach prioritizes robust sensor fusion and simulation-validated software before hardware integration.

Technical Architecture

The navigation system is built on ROS2 (Jazzy) using the Nav2 stack, chosen for its modular architecture and active development community. Key design decisions include:

  • Localization: Fusing wheel encoder odometry, IMU data, and depth camera input for state estimation in GPS-denied environments. Using robot_localization package with extended Kalman filter for sensor fusion.
  • Path Planning: Nav2's behavior trees for mission logic, allowing modular task sequencing (excavate → navigate → deposit → navigate → repeat).
  • Motor Control: REV Robotics Spark Flex controllers communicating over CAN bus, providing closed-loop velocity control with encoder feedback.
  • Simulation-First Development: Full Gazebo simulation environment mirroring competition arena, enabling software validation before hardware availability.

Current Status

The project is in active development ahead of the 2026 competition. The mechanical team is manufacturing the chassis and digging system while I continue software development in simulation. Current focus areas:

  • Completed: ROS2 architecture design, Gazebo simulation environment, CAN communication protocol for Spark Flex controllers
  • In Progress: Ordering the LiDARs, excavation system design and manufacturing, testing the camera/IMU
  • Upcoming: Hardware integration testing once excavation system is complete, full system validation on beach as a mockup arena

Technical Challenges

Key challenges we're addressing:

  • GPS-Denied Localization: The competition arena prohibits GPS, requiring robust odometry and visual localization. Wheel slip in loose regolith makes pure encoder odometry unreliable.
  • Dust Mitigation: Lunar regolith simulant is abrasive and electrostatically charged, posing challenges for sensors and mechanical systems.
  • Autonomous Decision Making: The rover must handle unexpected obstacles and terrain variations (craters) without operator intervention.
